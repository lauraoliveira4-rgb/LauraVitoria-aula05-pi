Criação do repositório
O que é

O repositório é basicamente o início do projeto, o local onde são armazenados arquivos importantes.

Sua funcionalidade

Ele pode ser armazenado tanto localmente quanto na nuvem, permitindo salvar o projeto e compartilhar com outras pessoas.

Nesse print é possível ver a tela de criação do repositório. Ele pode ser criado quando você entra no perfil ou na aba de criação do GitHub.

Clone do repositório
O que é

O clone do repositório serve para criar uma cópia do repositório remoto no computador local.

Sua funcionalidade

Esse clone é útil para salvar os arquivos da equipe e sempre deixar uma versão atual pronta para trabalhar ou caso algo dê errado.

O clone pode ser feito usando o GitHub Desktop.

Criação de branch

Podemos pensar em branch como galhos de uma árvore.
Ela é uma extensão alternativa do projeto, como se fossem multiversos do mesmo código.

Sua funcionalidade

A branch serve para organizar várias coisas ao mesmo tempo e permitir que vários membros trabalhem no mesmo projeto sem interferir diretamente no código principal.

<img width="450" height="275" alt="image" src="https://github.com/user-attachments/assets/36472c49-02e6-46e5-bb0b-06d8bbaa935a" />

Para fazer uma branch nova é bem simples. No seu repositório, ao lado de main, clique na opção branch. Depois clique em New Branch.

Registro de commits
O que é e sua funcionalidade

Commit é o ato de salvar alterações feitas no projeto.

Ele permite rastrear mudanças, identificar bugs e até voltar o código para um estado anterior se algo der errado.

Também é importante usar mensagens de commit claras, explicando o que foi alterado.

Push e Pull
Push

O push envia as alterações feitas no computador local para o repositório remoto no GitHub.

Ou seja, ele atualiza o repositório online com as mudanças feitas.

Dica: para fazer um push é necessário primeiro fazer um commit.

Pull

O pull serve para baixar ou atualizar as alterações do repositório remoto para o computador local.

Assim todos da equipe conseguem trabalhar sempre com a versão mais atual do projeto.

Issues

As issues são usadas para registrar tarefas, problemas, melhorias ou ideias dentro do projeto.

Elas ajudam a organizar o trabalho da equipe e acompanhar o que precisa ser feito.

Podem ser criadas dentro do próprio repositório.

Pull Request

O Pull Request é usado quando alguém faz alterações em uma branch e quer enviar essas mudanças para a branch principal (main).

Ele permite que outras pessoas revisem o código antes de aceitar as mudanças, ajudando a evitar erros.

No Pull Request é possível:

comentar alterações

revisar código

aprovar ou pedir mudanças

Merge

O Merge acontece quando as alterações de uma branch são juntadas com a branch principal.

Depois que um Pull Request é aprovado, o código pode ser integrado ao projeto principal através do merge.

Isso faz com que as mudanças passem a fazer parte da versão oficial do projeto.

Colaboradores do repositório

Uma funcionalidade importante do GitHub é poder adicionar colaboradores ao projeto.

Isso permite que colegas e professores possam:

visualizar o projeto

sugerir melhorias

contribuir com código

Para adicionar colaboradores basta ir em:

Settings → Collaborators → Add people
